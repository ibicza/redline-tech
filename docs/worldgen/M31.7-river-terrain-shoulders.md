# M31.7 River terrain shoulders

## Problem

The fitted river profile can sit above locally generated terrain when HydroRIVERS,
the DEM, and vanilla terrain disagree. The channel itself remains coherent, but the
old outer bank filled only a three-block surface shell. Air below that shell made
the river look like an aqueduct and left a vertical dirt wall at its edge.

## Root cause

`AtlasSurfaceMaterialPolisher.finishRiverBankColumn` owned both containment and
visual blending. It created a symmetric rim, then deliberately stopped filling a
few blocks below the cap. There was no wider river context outside the bank, so low
land could not approach the fitted profile gradually.

## Design

River sampling now exposes three non-empty geometric zones:

- `CHANNEL`: authoritative water and river biome.
- `BANK`: sealed containment around the water.
- `SHOULDER`: dry terrain context outside the bank.

`RiverSample.hasRiverData()` remains true only for channel and bank. The shoulder
therefore cannot override lake/open-water height selection, biome water context, or
river fluid guards. `hasTerrainContext()` is used only while selecting the best
overlapping river segment.

The bank cap is fixed at the section's `bankTopY` and its fill reaches existing
terrain. The shoulder raises only low, dry, replaceable columns. Its target descends
from `bankTopY` under a configurable slope envelope and is capped by a configurable
maximum raise. It never lowers high terrain.

The shoulder pass refuses lake, coast, and ocean classifications, refuses liquid
columns, and preflights every block before editing. Unknown solid blocks therefore
protect structures from burial. Work is column-local, deterministic from atlas
geometry and coordinates, and idempotent. Neighboring chunks do not exchange
mutable state, so generation order cannot change the result.

## Runtime and memory

The existing river spatial index expands segment bounds by the configured shoulder
width. No new cache or per-world retained collection is introduced. Per affected
column, the pass performs one terrain lookup and at most
`riverBankMaxRaiseBlocks` preflight/fill iterations. The profiler records this work
as `surfacePolish.riverTerrainShoulder`.

Defaults:

- `terrainShoulderWidthBlocks = 32`
- `riverBankMaxRaiseBlocks = 12`
- `riverTerrainShoulderMaxSlope = 0.5`
