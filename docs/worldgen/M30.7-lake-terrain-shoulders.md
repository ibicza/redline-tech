# M30.7 Lake terrain shoulders

## Problem

Residual WorldCover lakes use a fixed fitted water level and an authoritative
horizontal water mask. The existing `LAKE_SHORE` ring both contained that water
and flattened every low shore column to `waterY`. Its outer edge could therefore
end as a vertical sand wall. Deep raised columns were also made entirely from sand.

## Root cause

One classification owned three separate concerns: water containment, beach
material, and transition to surrounding terrain. The shore search was intentionally
tight to avoid broad random sand halos, so there was no dry context in which to
form a gradual transition.

## Design

The lake pipeline now keeps two land zones:

- `LAKE_SHORE` is the existing narrow containment and sand zone.
- `LAKE_TERRAIN_SHOULDER` is a wider, surface-only dry terrain context.

The shoulder is not lake data: its `hasLakeData` flag is false, it is never returned
by the public exact lake command sample, and biome/height/water logic ignores it.
The existing bounded biome-cell cache stores only a padded proximity hint. During
surface polish, an exact shoulder scan is performed only for hinted cells; the
padded hint itself is never allowed to edit terrain.

Low shoulder columns rise under a configurable slope envelope descending from the
contained shore. The pass has a separate maximum raise, never lowers high terrain,
refuses liquid columns and ocean/coast classifications, and preflights unknown
solid blocks before making changes. Work is deterministic, idempotent, and owned
by the current column, so chunk generation order does not affect the result.

The narrow containment bank keeps its existing raise cap. It now preflights the
whole column before filling, preserves the natural cap when slope rules reject
sand, and uses sandstone/stone/dirt below the configured surface depth. Falling
sand therefore always has a stable foundation instead of forming a deep sand wall.

Defaults:

- `terrainShoulderRadiusBlocks = 32`
- `lakeTerrainShoulderMaxRaiseBlocks = 12`
- `lakeTerrainShoulderMaxSlope = 0.5`

## Runtime and memory

No unbounded collection or new cache is introduced. The existing bounded
`SAMPLE_CACHE` provides coarse hints at its configured cell size and clear-all
limit. Far land performs no per-column wide shoulder scan. Near a hinted lake,
surface polish performs one exact bounded scan and at most the configured maximum
raise worth of preflight/fill operations. Timing is recorded as
`surfacePolish.lakeTerrainShoulder`.
