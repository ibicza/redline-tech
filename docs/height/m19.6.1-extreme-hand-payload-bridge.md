# M19.6.1 — Extreme hand payload bridge

M19.6 fixed the server-side void kill for the true cubic vertical range, but normal hand interaction at Y=9000/-12000 could still be swallowed by vanilla client/server interaction paths before the cube backend was mutated.

This patch adds a temporary native interaction payload for cube-only blocks outside the vanilla shell:

- left click on an outside-shell cube block sends `CubicExtremeInteractionPayload.BREAK`;
- right click with a `BlockItem` sends `CubicExtremeInteractionPayload.PLACE`;
- the server validates that the target is inside the internal cubic range and outside the temporary shell;
- the server mutates the cube backend, saves the cube, sends a native delta, and queues a repair snapshot.

The normal vanilla packet bridge remains as a fallback, but the client payload is now the main path for outside-shell hand interaction.

Still not solved here: real block-model rendering outside the vanilla shell. That remains the next native renderer milestone.
