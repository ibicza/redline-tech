# M19.5 — native extreme-height interaction probes

Цель этапа — проверить следующий слой после M19.4: не только хранить кубы вне временного vanilla shell, но и готовить игрока, collision, raycast и native section sync к работе на высотах вроде `Y=9000` и `Y=-12000`.

## Важное ограничение

M19.5 ещё не является полноценным native cube renderer. Вне vanilla shell блоки уже живут в cube backend, collision/query могут их видеть, native section snapshots отправляются клиенту, но финальная отрисовка блок-моделей вне vanilla render sections всё ещё остаётся следующим большим этапом.

Vanilla shell остаётся временной оболочкой только для диапазона `-2032..2031`. Всё, что выше/ниже, должно идти только через cube backend/native section path.

## Новые команды

### `/rwc cubic_test teleport_extreme high`

Создаёт каменную платформу на `Y=9000`, чистит воздух над ней, сохраняет затронутые cube, телепортирует игрока и принудительно отправляет native snapshots затронутых секций.

### `/rwc cubic_test teleport_extreme low`

То же самое, но на `Y=-12000`.

### `/rwc cubic_test teleport_extreme <y>`

Пользовательский вариант в пределах internal cubic range.

### `/rwc cubic_test collision_probe`

Проверяет:

```text
- позицию игрока;
- playerCube;
- onGround;
- delta movement;
- блок в ногах;
- блок под игроком;
- cube под игроком;
- был ли принудительно отправлен native snapshot.
```

Если блок под игроком не air, серверный cube-backed collision слой видит опору.

### `/rwc cubic_test native_raycast`

Серверный native raycast по cube backend из глаз игрока. Не зависит от vanilla shell height.

Показывает:

```text
hit block
previous air block
cube position
block state
distance
samples
```

### `/rwc cubic_test native_snapshot current`

Принудительно отправляет клиенту native snapshot текущего cube игрока.

### `/rwc cubic_test native_snapshot look`

Принудительно отправляет native snapshot cube, найденного server-side native raycast-ом.

### `/rwc cubic_test native_interact break`

Debug-ломание блока через cube backend по server-side native raycast. Работает вне vanilla shell, сохраняет cube и отправляет fresh native snapshot.

### `/rwc cubic_test native_interact place <block>`

Debug-постановка блока через cube backend в последний air перед raycast-hit. Работает вне vanilla shell, сохраняет cube и отправляет fresh native snapshot.

### `/rwc cubic_test materialize_extreme high|low`

Проверяет, что вне vanilla shell мы НЕ пытаемся писать блоки во временный vanilla `ClientLevel/ServerLevel` shell. Команда должна завершиться ожидаемым blocked-result.

## Что тестировать вручную

1. Выполнить:

```text
/rwc cubic_test teleport_extreme high
/rwc cubic_test collision_probe
/rwc cubic_test native_raycast
```

2. Посмотреть, стоит ли игрок стабильно на платформе или его дёргает.

3. Выполнить:

```text
/rwc cubic_test native_interact break
/rwc cubic_test native_interact place glass
/rwc cubic_test native_snapshot current
```

4. Повторить то же для low:

```text
/rwc cubic_test teleport_extreme low
/rwc cubic_test collision_probe
/rwc cubic_test native_raycast
```

## Ожидаемый результат M19.5

```text
storage/query: работает вне vanilla shell
server collision: должен видеть cube blocks вне shell
server native raycast: должен находить cube blocks вне shell
debug place/break: должен менять cube backend вне shell
native snapshots: должны принудительно уходить клиенту
vanilla materialize: должен быть запрещён вне shell
```

## Что останется после M19.5

```text
M19.6+ native cube renderer:
  реальная отрисовка BlockState-моделей вне vanilla render sections

M19.7+ native client interaction:
  обычный ЛКМ/ПКМ игрока через native raycast/path, а не debug-команды

M19.8+ outside-shell gameplay polish:
  selection outline, particles, sounds, lighting, entity tracking на extreme height
```
